# Project Structure

This document keeps the detailed repository structure in one place for both humans and agents.

## Root Overview

```text
.
├── backend/                 # FastAPI backend application
├── frontend/                # Frontend source/build setup
├── static/                  # Generated frontend assets served by backend
├── docs/                    # Documentation
├── compose.yaml             # Local development with Docker Compose
├── Dockerfile               # Container image definition
├── {extid}_identity.json    # Extension identity/auth (generated by instance bootstrap use the automatic external id, git ignored)
├── meta.yaml                # Extension metadata template
├── settings.yaml            # Public non-sensitive settings
├── .secrets.yaml            # Private sensitive settings (git ignored)
├── LICENSE.txt
└── README.md
```

## Backend Structure (Detailed / Stable)

```text
backend/
├── pyproject.toml           # Python dependencies and metadata
├── uv.lock                  # Locked Python dependency versions
└── app/
    ├── __init__.py
    ├── main.py              # Extension bootstrap and CLI entry point
    ├── extension.py         # FastAPI app setup, router mounting, and static serving
    ├── routers/             # FastAPI routers, one file per endpoint category
    │   ├── __init__.py
    │   ├── api.py           # Authenticated REST endpoints  (prefix /api/v1)
    │   ├── bypass.py        # Unauthenticated / internal endpoints  (prefix /bypass)
    │   ├── events.py        # Platform event handlers  (prefix /events)
    ├── auth.py              # Authentication context management
    ├── client.py            # Marketplace API client
    ├── config.py            # Settings loading and config handling
    ├── logging.py           # Logging tools and configuration
    ├── schema.py            # Pydantic request/response models
    └── utils.py             # Shared utility functions
```

## Frontend Structure (Flexible / Evolving)

The frontend file list is intentionally not strict and may change over time.
Use this as a baseline, not a fixed contract.

```text
frontend/
├── package.json             # Scripts and frontend dependencies
├── tsconfig.json            # TypeScript config
├── esbuild.config.js        # Bundling config
└── src/
    ├── index.tsx            # Current entry point
    ├── App.tsx              # Current root component
    └── style.scss           # Current global styles
```


## Static Assets

`static/` contains build artifacts produced by the frontend build process and served by the backend.
